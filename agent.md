## Issue Creation Guidelines

When creating GitHub issues:
- Always follow the standard issue structure
- Ensure issues are reproducible and actionable
- Clearly identify affected components using repository terminology
- Include expected vs actual behavior
- Reference relevant modules or directories when possible

## General Behavior

- Ask for clarification if information is missing
- Do not assume missing technical details
